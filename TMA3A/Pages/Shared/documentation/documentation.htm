Part 3


Link to Part 3: https://tma3anew.azurewebsites.net/part3/part3



Description of the task and requirements:


In this part of the assignment, the main objective was to develop an online computer webstore using ASP.NET in C# in which customers can select a computer from a list and customize it by replacing some components in its standard configuration. Customers can then place an order with the online store. This is what is stated in the assignment description.


The requirements were quite specific for this task:

  • The application running in a web browser should be like a real online store.
  • The application has a clean and easy-to-use interface, with menus and navigation buttons
  • It has a name and logo shown across all the pages
  • It begins with a welcome message to greet customers
  • It has other pages such as policies, a feedback form, etc
  • There should be at least 5 components available
  • A customer can select a computer with default configuration and customize it.
  • Whenever a change has been made by the customer, the price total updates dynamically
  • After the customer has hit the Buy button, a summary is displayed.
  • All links, menus, and buttons within the online store should be active. .


Analysis and Design of the Web Application:


As asked in the assignment description, I visited some online stores such as Bestbuy.ca and Apple.ca and analysed their designs/functionality. Here’s what I found:

  • At the very top, all these websites had a menu bar with navigation links to the main pages of the website.
  • The navbar included a button to manage your account and a button to check your cart on the far right, and a search bar in the center
  • Some websites had a welcome message, a slideshow with their latest products, along with some nice graphics
  • When we go onto their catalog, we observe a picture of the product displayed along with the price, and a button to add the item to the cart.
  • On some websites, there was an additional button to view details of the product before adding it to cart.
  • When items are added to cart, we observe that following: a picture of the product, the option to remove the item, its quantity, and its price.
  • At the very bottom we observe the grand total
  • When the buy button is clicked, we see a summary of our items and some input boxes to enter in our personal information.
  • Once an order is submitted, a confirmation email is recieved
  • Some websites had a feedback form

These websites were carefully studied and analyzed to create my web application. I created rough storyboards of the web app’s functionality and implemented it using Agile methodology and daily sprints. Pretty much all of the observations listed above are implemented into my website. Below I will provide you with my rough storyboards I made before I started coding the web app and the final results as well!
